2. 결정사항



	현재 개발 중인 프로토 타입 데이터 관리 방법
	웹 상의 크롤링 후 로컬에 저장하여 데이터 관리
	db 환경 또한 localhost 사용으로 인터넷 상이나 클라우드 상의 공유 불가
	각기 다른 컴퓨터에 같은 개발환경을 구축해야하고 sql 파일을 계속 공유해야하는 번거로움 발생
		서버용 하드웨어 기기 마련 시 구글 드라이브도 어느정도 대체 가능
		데이터 클라우드화 시 모든 사람들이 접속 가능하지만 보안의 증대 필요

	발전 예상 데이터 관리 방법
	1. RDS 사용 
		aws 사의 RDS 를 사용하여 각 로컬에서 관리되는 db나 sql 파일들을 실시간으로 공유하고 계속 sql 파일을 저장하고 공유해야하는 번거로움을 줄임
		초기 및 유지 보수 비용 발생

	2. 구글 시트 사용
		데이터의 형태가 데이터 프레임(테이블)의 형태를 아직 벗어나지 않기 때문에 구글 드라이브의 시트를 통한 공유 가능
		강력한 데이터 무결성 유지 불가 (만약 구글 드라이브의 로딩 상태에서 데이터 변형 가능성 있음 -> 실시간 적용이 잘 이루어지지 않는 경우 발생 시 데이터 유지에 치명적
		용량 제한 -> 애초에 엑셀 기반의 작업이라 엑셀과 마찬가지로 데이터 용량의 한계와 성능 퍼포먼스 저하를 유발할 수 있음

	3. 하이브리드 파이프라인 개발
		구글 드라이브에 sql 파일 자체를 저장해서 계속해서 공유하는 방법
		데이터에 직접 접근하려는 컴퓨터를 지정해서 해당 컴퓨터에만 파이프라인 구축하여 데이터 분석
		그 후 바로 bi 변환 가능한 최종 데이터 가공 파일로 구글 드라이브에 저장하거나, 
		아예 시각화 된 파일을 저장

3. 테이블 컬럼 지정
	일단 사용하고 분석할 report 파일 종합 및 각 크롤링 자동화 파이프라인 코드 개발
	 report  별 컬럼 구분 및 지정
	원하는 시각화 포인트
		PowerBI 선택
		시각화 종류에 대해 분석 및 장단점 조사
