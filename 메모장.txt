주당 100KB 기준 연간 데이터 양:

1년 = 52주
연간 데이터 양 = 100KB * 52 = 5,200KB ≈ 5.08MB


Dashboard_raw Data 분석:

현재 크기: 21,000KB ≈ 20.5MB
기간: 2023년 3월 ~ 2024년 5월 (약 14개월)
월평균 데이터 양: 20.5MB / 14 ≈ 1.46MB/월
분기별 데이터 양: 1.46MB * 3 ≈ 4.38MB/분기


DB 전환 시 데이터 압축:

일반적으로 엑셀에서 정규화된 DB로 전환 시 30-50% 정도 데이터 크기가 줄어들 수 있습니다.
보수적으로 30% 압축을 가정하면:

분기별 데이터 양: 4.38MB * 0.7 ≈ 3.07MB/분기
연간 데이터 양: 3.07MB * 4 ≈ 12.28MB/년





이 정도의 데이터 양으로는 여전히 Spark의 필요성이 크지 않습니다. 그러나 다음 사항들을 고려해볼 필요가 있습니다:

데이터 증가율:
현재 속도로 증가한다면 5년 후에는 약 61.4MB의 연간 데이터가 생성될 것입니다. 이는 여전히 Spark가 필요한 수준은 아닙니다.
데이터 복잡성:
데이터의 양뿐만 아니라 복잡성도 중요합니다. 복잡한 분석이나 머신러닝이 필요하다면 Spark의 도입을 고려해볼 수 있습니다.
실시간 처리 요구:
실시간 데이터 처리가 필요하다면 Spark Streaming을 고려해볼 수 있습니다.
확장성:
비즈니스가 급격히 성장하여 데이터 양이 예상보다 빠르게 증가할 가능성이 있다면, 미리 Spark를 도입하는 것도 방법입니다.
기술 스택 통일:
회사의 다른 프로젝트나 부서에서 이미 Spark를 사용하고 있다면, 기술 스택을 통일하기 위해 도입을 고려해볼 수 있습니다.

결론:
현재의 데이터 양과 증가 추세만을 고려한다면, Spark의 필요성은 여전히 낮습니다. 그러나 데이터의 복잡성, 실시간 처리 요구, 미래의 급격한 성장 가능성, 회사의 기술 전략 등을 종합적으로 고려해야 합니다.
대안으로 다음과 같은 접근을 제안합니다:

현재 시스템 최적화: Python과 MySQL을 사용하여 시스템을 최적화합니다.
중간 규모의 솔루션 도입: Dask나 Vaex와 같은 Python 기반의 대용량 데이터 처리 라이브러리를 도입합니다. 이들은 Spark보다 진입 장벽이 낮고, 기존 Python 코드와의 호환성이 좋습니다.
모니터링 및 평가: 데이터 증가율과 처리 요구사항을 지속적으로 모니터링합니다.
단계적 도입 계획: 향후 2-3년 내에 Spark 도입이 필요할 것으로 예상된다면, 미리 팀 교육과 소규모 파일럿 프로젝트를 통해 준비를 시작합니다.

이러한 접근 방식을 통해 현재의 요구사항을 충족하면서도 미래의 확장성을 준비할 수 있을 것입니다.

월평균 1.46MB의 데이터를 3년 동안 처리하고 저장하는 시스템을 위한 하드웨어 스펙을 제안해드리겠습니다. 이는 중소규모의 데이터 처리 및 분석 워크로드를 가정한 것입니다.

총 예상 데이터 양:

월 1.46MB * 36개월 = 52.56MB (약 53MB)
이 정도의 데이터 양은 현대적인 하드웨어 기준으로 매우 작은 편이지만, 데이터 처리, 분석, 백업 등을 고려하여 여유 있게 스펙을 산정하겠습니다.

권장 하드웨어 스펙:

CPU:
4코어 8스레드 프로세서 (예: Intel Core i5 또는 AMD Ryzen 5 시리즈)
클럭 속도: 3.0GHz 이상
RAM:
16GB DDR4
(데이터 전체를 메모리에 올리고 처리하기에 충분)
스토리지:
운영체제 및 애플리케이션용: 256GB SSD
데이터 저장용: 1TB SSD 또는 HDD
(실제 데이터는 작지만 백업, 로그, 임시 파일 등을 고려하여 여유 있게 설정)
네트워크:
기가비트 이더넷 (1Gbps)
운영체제:
Linux (Ubuntu 20.04 LTS 또는 CentOS 8) 또는 Windows Server 2019
백업 시스템:
외장 HDD 또는 네트워크 연결 스토리지(NAS) 1TB
추가 고려사항:

확장성:
향후 데이터 증가에 대비해 RAM과 스토리지를 쉽게 업그레이드할 수 있는 시스템 선택
안정성:
RAID 구성을 통한 데이터 안정성 확보 (RAID 1 또는 RAID 5 권장)
전원 공급:
무정전 전원 공급 장치(UPS) 사용 권장
냉각:
적절한 냉각 시스템 확보
모니터링:
시스템 리소스 모니터링 도구 설치 (예: Nagios, Zabbix)
이 스펙은 현재 예상되는 데이터 양과 처리량을 충분히 커버할 수 있으며, 향후 2-3년간의 완만한 성장도 수용할 수 있습니다. 그러나 데이터 양이 급격히 증가하거나 복잡한 실시간 분석이 필요해지면 스펙 업그레이드가 필요할 수 있습니다.

또한, 클라우드 서비스 (예: AWS, Azure, GCP)를 고려해볼 수 있습니다. 클라우드를 사용하면 초기 하드웨어 투자 비용을 줄이고 필요에 따라 쉽게 스케일업/다운할 수 있는 장점이 있습니다.
